# /checkpoint コマンド

`.steering/` を棚卸しして進行中・完了・放置中のフィーチャーを一覧表示するコマンド。
セッションをまたいで作業を再開するときや、プロジェクト全体の進捗を把握したいときに使う。

**使い方**: `/checkpoint`  
**目的**: 「今どこまで進んでいるか」を即座に把握し、次のアクションを決める

---

## 重要ルール

- **このコマンドはファイルを変更しない**（読み取り専用）
- 完了済みフィーチャーのアーカイブはユーザーの確認を取ってから行う

---

## ステップ 1: .steering/ のスキャン

`.steering/` ディレクトリ以下のフィーチャーディレクトリを列挙する。

```bash
ls -lt .steering/
```

各フィーチャーディレクトリの `tasklist.md` を読み、以下の基準でステータスを判定する:

| ステータス | 判定基準 |
|---|---|
| ✅ 完了 | 全タスクが `[x]`、かつ振り返りセクションが記入済み |
| 🔄 進行中 | `[x]` と `[ ]` が混在している |
| 🔲 未着手 | 全タスクが `[ ]` |
| ⚠️ 放置中 | `[ ]` が残っているが最終更新から日数が経過している |

---

## ステップ 2: 状態別一覧を表示

以下のフォーマットで報告する:

```
## チェックポイント — {今日の日付}

### 🔄 進行中
- {YYYYMMDD}-{feature-name}
  進捗: {完了数}/{全タスク数} タスク完了
  直近のタスク: {最後に [x] になったタスク名}
  残タスク: {最初の [ ] のタスク名}

### 🔲 未着手
- {YYYYMMDD}-{feature-name}
  作成日: {YYYY-MM-DD}

### ⚠️ 放置中（14日以上更新なし）
- {YYYYMMDD}-{feature-name}
  最終更新: {YYYY-MM-DD}（{N} 日前）
  残タスク: {[ ] の数} 件

### ✅ 完了（直近 3 件）
- {YYYYMMDD}-{feature-name} — {振り返りの1行目}
- ...

---
進行中: {N} 件 / 放置中: {N} 件 / 完了: {N} 件
```

---

## ステップ 3: 次のアクション提案

表示後、以下を提案する:

**進行中のフィーチャーがある場合**:
```
🔄 {feature-name} を再開しますか？
  → 再開する場合は `/add-feature {feature-name}` を実行してください
     （既存の tasklist.md を読み込んで続きから始まります）
```

**放置中のフィーチャーがある場合**:
```
⚠️ 放置中のフィーチャーがあります。以下のいずれかを選んでください:
  1. 再開する → `/add-feature {feature-name}`
  2. キャンセル扱いにする → tasklist.md に理由を記録して完了マーク
  3. そのまま放置
```

**すべて完了している場合**:
```
✅ すべてのフィーチャーが完了しています。
  次のフィーチャーを始める場合は `/add-feature {機能名}` を実行してください。
```

---

## ステップ 4: docs/ との整合性チェック（オプション）

完了済みフィーチャーが存在する場合、以下を確認してユーザーに報告する:

- `docs/product-requirements.md` のフェーズステータスに反映されているか
- `CLAUDE.md` の「実装済み」セクションに追記されているか

ズレがある場合:
```
📝 以下のドキュメントが最新化されていない可能性があります:
  - docs/product-requirements.md（フェーズ {N} のステータスが未更新）
  - CLAUDE.md（実装済みセクション）
  → `/update-docs` を実行すると最新化できます
```
