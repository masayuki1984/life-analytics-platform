# 要件定義: Daily Note テンプレート対応

**作成日**: 2026-06-22

---

## 概要

現在、録音が 0 件の日は Daily Note が空ファイル（0 bytes）で作成される。
今後は `{OBSIDIAN_VAULT_PATH}/Templates/daily.md` をテンプレートとして Daily Note を生成し、
録音の有無にかかわらず Plaud セクションに適切な内容を書き込む。

## スコープ

### IN（実装する）

- Daily Note 新規作成時に `Templates/daily.md` の内容をコピーする
- テンプレートが存在しない場合は警告ログを出して空ファイルにフォールバックする（既存挙動）
- 録音がない場合、Daily Note の `## Plaud` セクションに `録音なし` と書き込む
- 録音がある場合は既存通り Plaud セクションにリンクを追記する
- 既存の Daily Note は上書きしない（ファイルが存在すれば内容を保持）
- `## Plaud` セクションがなければファイル末尾に追加する（既存挙動）
- `録音なし` / リンクの重複書き込みは行わない（冪等性）
- `--dry-run` でも同じロジックを適用する（ファイルは書かない）
- テストを追加・更新する
- README / docs にテンプレート利用の旨を追記する

### OUT（実装しない）

- テンプレート内の変数展開（`{{date}}` 等）— Obsidian 側の機能に委ねる
- テンプレートパスの環境変数による設定変更
- 既存 Daily Note への遡及的なテンプレート適用

## 機能要件

### 主要機能

1. **テンプレートコピー**: `_ensure_daily_note()` で新規作成時に `Templates/daily.md` が存在すれば内容を `write_text` でコピーする
2. **録音なし書き込み**: `write_no_recordings(d)` パブリックメソッドを追加し、Plaud セクションに `録音なし` を追記する
3. **冪等性**: `録音なし` が既に書き込まれていれば何もしない

### データ要件

- **入力**: `{vault_path}/Templates/daily.md`（テンプレートファイル）
- **出力**: `{vault_path}/Daily/YYYY/MM/YYYY-MM-DD.md`（既存仕様通り）
- **保存先**: 変更なし

## 非機能要件

- **エラーハンドリング**: テンプレートが存在しない場合は `logger.warning` を出して空ファイルで続行
- **ログ**: 新規作成時にテンプレート使用か否かをログに記録する
- **dry-run**: 実ファイルを読み書きしない（テンプレートの存在チェックのみ行う）

## 受け入れ条件

- [ ] テンプレートが存在する場合、新規 Daily Note にテンプレート内容がコピーされる
- [ ] テンプレートが存在しない場合、空ファイルで作成される（WARNING ログあり）
- [ ] 録音 0 件の日、Daily Note の `## Plaud` セクションに `録音なし` が書き込まれる
- [ ] `録音なし` を 2 回実行しても重複しない
- [ ] 既存 Daily Note がある場合は内容を保持し、Plaud セクションへの追記のみ行う
- [ ] `--dry-run` でファイルが作成・変更されない
- [ ] `pytest` が全 GREEN
- [ ] `ruff check .` が通る
- [ ] README に `Templates/daily.md` の利用について記載がある
